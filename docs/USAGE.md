# LayCAT 使い方ドキュメント

映像制作向けのレビュー／進行管理ツール（ShotGrid ライク）。単一 HTML（`laycat.html`）で動きます。
本ドキュメントは基本操作と、**対応する動画コーデック**（重要）をまとめたものです。概要は `README.md` も参照してください。

- 本体：https://ogshaw03.github.io/laycat/
- アクセス管理コンソール：https://ogshaw03.github.io/laycat/access-console.html
- 推奨ブラウザ：**Chrome / Edge**（フォルダ共有に File System Access API が必要）

---

## 1. はじめかた

1. 本体URLを開く（ライブサイトでは Google ログイン。ローカル/exe/プレビューは認証オフ）。
2. **新規プロジェクト作成**時に**保存先フォルダを指定**する。
   - 未設定だと警告が出て、メタ情報は一時的にブラウザ内（localStorage）に保存され、**動画・画像はフォルダ設定まで保存できません**。
   - 保存先はファイルサーバー上のフォルダを指定すると、それがチームの「1つの正」になります。
3. 他の人が作ったプロジェクトは、ホームの **「⇄ プロジェクトを接続」** でそのフォルダを開くと自分の一覧に追加されます。

保存先の内訳：
- プロジェクトデータ：フォルダ内 `laycat.project.json`
- メディア：フォルダ内 `media/`
- リール：フォルダ内 `reels/reels.json`
- 個人設定・接続一覧・通知：各自のブラウザ内（localStorage、共有データとは分離）

---

## 2. アップロードとレビュー

1. ショット／工程（タスク）を開いて動画・画像をアップロード（ドラッグ&ドロップ、またはアップロードボタン）。
2. 動画レビュー画面で**フレーム単位のコメント／アノテーション**（ブラシ・レイヤー・トレーシングペーパー・ゴースト表示・顔の向きガイド）。
3. ディレクターが **OK / リテイク / NG / 保留** を判定。
4. **⇄ 比較再生（A/B）**：別の版・カットと横並び同期再生。
5. **@名前 / @ロール** でメンション → ベルに通知。

---

## 3. 工程（タスク）と進捗

- ショットごとの各工程を「未着手 / 作業中 / 確認待ち / リテイク / OK」で管理。作業担当・チェック担当を工程単位で設定。
- タブ：タイムライン（更新履歴）／進捗／ショット（ホバー再生）／サブミット／チェック待ち。
- **サブミット**：複数カットをまとめて提出。提出ページに固有URLを添付可能。
- **REEL**：動画クリップを並べて連続再生・提出。

---

## 4. 画面ごとの共有URL

各タブ／ページは固有のURL（ハッシュ）を持ち、共有すると相手が同じ画面を開けます。

- `#/n/<プロジェクトID>/<タブ>` … タブ（timeline / progress / shots / submit / check / browse）
- `#/r/<ノードID>/<バージョンID>` … 動画レビュー（アノテーション）画面
- `#/submit/<サブミットID>` … 提出ページ

---

## 5. アクセス管理・メンテナンス・お知らせ（運営／管理者）

**アクセス管理コンソール**（`access-console.html`）から操作します。

- **アクセス管理**：認証必須トグル、許可メール／ドメイン、権限（運営 / 管理者 / メンバー）、招待リンク（承認不要URL）の発行・無効化。
- **🛠 メンテナンス制御**：開始すると全メンバーがメンテナンス画面に。運営・管理者は上部バナーで閲覧可。**解除すると全メンバーのページが自動的に再読み込み**されます。
- **📢 全メンバーへのお知らせ**：メッセージを全メンバーの画面上部にバー表示。各メンバーは「確認して閉じる」で自分の端末だけ消せます。

> メンテナンス／お知らせは Firestore（`laynaAccess/maintenance`・`laynaAccess/broadcast`）で共有します。**利用には、コンソールのルール欄に表示される Firestore セキュリティルールを Firebase コンソールへ反映**しておく必要があります（未反映だと開始／送信が権限エラーになります）。

## 6. 対応する動画コーデック（.mov / .mp4）

`.mov` ファイル自体は受け入れます（MIME が空でも拡張子で動画と判定）。ただし**サムネイル生成・再生ができるかは中の「コーデック」次第**です。ブラウザ（`<video>` タグ）が再生できるのは **H.264 / H.265 / VP9 / AV1** などに限られ、これは LayCAT に限らずどの Web ツールでも共通の制約です。

### Maya の QuickTime（qt）書き出しコーデック別の対応

| Maya QuickTime の書き出しコーデック | ブラウザでの再生 | LayCAT での表示 |
| ---- | ---- | ---- |
| **H.264** | Chrome / Edge / Safari で可 | ✅ サムネ・再生OK |
| Animation（RLE） | 全ブラウザ不可 | ❌ 保存はできるが黒画・再生不可 |
| Photo-JPEG | 全ブラウザ不可 | ❌ 同上 |
| PNG / None(raw) | 全ブラウザ不可 | ❌ 同上 |
| Apple ProRes | 全ブラウザ不可 | ❌ 同上 |

> Maya の QuickTime 書き出しは Animation・Photo-JPEG・ProRes などが選ばれていることが多く、その場合はブラウザで再生できません。

### 確実に動かすには（H.264 で書き出す／変換する）

- **Maya 側**：Playblast／ムービー書き出しのエンコーダを **H.264** に設定する。
- **変換**：
  ```
  ffmpeg -i input.mov -c:v libx264 -pix_fmt yuv420p -movflags +faststart output.mp4
  ```
  （`-pix_fmt yuv420p` が重要。422/444 だとブラウザで再生できないことがあります）

> 再生できないコーデックの動画は、LayCAT の再生窓の中に「この動画はブラウザで再生できないコーデックの可能性があります（Animation / ProRes 等）。H.264 の mp4 / mov へ変換してください」と常時表示されます。

## 7. プロジェクトの暗号化（任意）

各プロジェクトに個別パスワードを設定でき、パスワードから導出した鍵（PBKDF2→AES-GCM）でプロジェクトデータ（コメント・ノート等）を暗号化して保存します。パスワード紛失時は復旧不可（※動画・画像ファイル本体は暗号化対象外）。詳細は `docs/PROJECT_ACCESS_PLAN.md`。
